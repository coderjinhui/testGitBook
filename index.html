Introduction

简介

Angular 是一个相当强大的框架，很多时候我们甚至不需要额外引入任何东西就能完成一个完整的前端项目。但也正是它的强大限制了它的发展，框架的复杂度竟能达到 "谈 Angular 色变" 的程度。

在我看来，Angular 的难度有些被人过于解读，在我决定学 Angular 之前就遇到过这么一次经历:

我【惊奇】: 好像 Angular 看上去挺厉害的样子。
小明【嫌弃】: 难到爆炸了! 乱七八糟的概念多到想不到!
我【犹豫】: 有多难? 最近我想学一下。
小明【小声 bb】: 别吧，这玩意的难度不是人学的。
我【期待】: 嗷。。。你学过吗，那么难要不你教我一下?
小明: 我没学过啊，这玩意这么难学了干啥?
我: 没学过你怎么知道它难啊?
小明【理直气壮】: 网上都这么说的呀!
我: 。。。

相对于其他框架来说，Angular 确实会难不少，但从一整个项目的角度来说其实难度和其他框架并没有相差太多。只不过 React 和 Vue 更偏重核心功能，而 Angular 更偏重大而全。于是在感官上好像确实 React 和 Vue 学起来简单点，因为人们并没有将其他的插件学习时间计算进去。

特性

框架之间的比较就不做过多解读，这容易引起部分忠实粉丝的反感，也容易被人过于解读。

事实上，框架是用来解决具体问题的，它只是我们手中的工具。就像吃饭，并不能说筷子就一定比刀叉好。在我看来，能吃进嘴就是完美的吃饭工具。每一个能够流行起来的工具必然有着它不可忽视的亮点。

Angular 有很多相当棒的特性，包括:

  • 跨平台
    • 支持构建 web 网页应用
    • 支持构建原生移动端应用
    • 支持构建桌面端应用
  • 高效
    • 强大的 Angular CLI
    • 支持多种 SSR
    • 代码拆分、按需加载
  • 大而全
    • 内置大量好用的模块，例如: 路由、网络请求等
    • 集成测试 ...等各种好用的特性

共勉

Angular 是一种 MVC 模式的框架，很多时候我们能在它身上看到后端思想的影子。但正如我之前说的，能解决问题的就是好工具。但也正是它身上的一些后端影子，虽然用起来确实会方便不少，但在学习上同样也增加了理解负担。这同样也是 "Angular 从入门到放弃" 的原因之一。

虽然我一直强调 Angular 的难度是被过于解读，但不可否认的是: 学习从来就不是一件容易的事，无论是学新的技术还是新的技能。而在学习过程中，主观能动性大于一切。这意味着如果你不想学，那么多半是怎么学都没用的。

至于为什么要学 Angular，倒不是真因为它多厉害才学它，当然这也是一部分原因。最主要还是归结于技多不压身。本领总归是越多越能提高自己的竞争力的。

写在后面

其实我也对 Angular 不是很熟，但为什么要写它？从个人角度来说其实是想记录下自己的学习成果，并且用它来对抗自己的遗忘。虽然我也可以用笔记的形式来做这些，事实上我也正是这么做的，但最后发现知识点相当散乱，完全不成体系。有时候我都怀疑我的笔记本是不是一个 Stack Overflow。因此狠下心来梳理自己的笔记，顺便试着把它记得不那么乱。

当然，如果哪天有读者能看到这些笔记，希望看过的人能心里发出一阵赞美: 哟嚯，666！
